CONTROLLED NOISE SIMULATION OF SVM-BASED BAYBAYIN CHARACTER RECOGNITION
========================================================================

SECTION III: METHODOLOGY (Publication-Grade Version)

A. Research Design

This study employs a quantitative experimental research design centered on robustness evaluation of a 
Support Vector Machine (SVM) classifier for handwritten Baybayin character recognition under controlled 
noise conditions. The research framework is grounded in the following principles:

1. Systematic Experimentation: The study introduces controlled image degradation to evaluate model 
   behavior under realistic conditions, rather than relying solely on clean, curated datasets.

2. Quantitative Evaluation: Performance is measured across multiple dimensions (accuracy, precision, 
   recall, F1-score) at varying noise intensities to provide comprehensive robustness assessment.

3. Reproducibility: All methodological steps, hyperparameter selections, and noise injection parameters 
   are formally defined to ensure study reproducibility.

The overall experimental workflow follows this sequence:

   Data Acquisition → Preprocessing → Feature Extraction → Model Training → Hyperparameter Optimization
   ↓
   Noise Simulation (varying intensities) → Model Evaluation → Robustness Analysis → Error Analysis 
   ↓
   Visualization & Interpretation

B. System Architecture Overview

The proposed system implements a 7-module machine learning pipeline designed for both classification and 
robustness evaluation:

MODULE 1: Dataset Acquisition and Preprocessing
   Purpose: Standardize input data for consistent feature extraction
   Input: Raw Baybayin character images
   Output: Normalized, grayscale images (64×64 pixels)
   
   Processing steps:
   • Grayscale conversion (if RGB input)
   • Spatial resizing to uniform dimensions (64×64)
   • Intensity normalization to [0, 1] range
   • Binary thresholding using Otsu's method

MODULE 2: HOG Feature Extraction
   Purpose: Convert pixel-based images into numerical feature vectors suitable for SVM
   Input: Preprocessed images (64×64 pixels)
   Output: Feature vectors (1764 dimensions per image)
   
   HOG Configuration:
   • Pixels per cell: 8×8
   • Cells per block: 2×2
   • Number of orientations: 9
   • Block normalization: L2-Hys (L2-norm with clipping)
   
   HOG Formula (Orientation Histogram):
   For each cell c in the image:
       h(θ) = Σ mag(p) for all pixels p in cell c where orientation ≈ θ
   
   where θ ∈ {0°, 20°, 40°, ..., 160°} (9 orientation bins)
   and mag(p) is the gradient magnitude at pixel p.

MODULE 3: SVM-RBF Training with Hyperparameter Tuning
   Purpose: Learn optimal decision boundaries in feature space
   Algorithm: Support Vector Machine with Radial Basis Function (RBF) kernel
   
   SVM Formulation:
   Decision function: f(x) = sign(Σ α_i y_i K(x_i, x) + b)
   
   where:
   • K(x_i, x) is the RBF kernel: K(x_i, x) = exp(-γ ||x_i - x||²)
   • α_i are support vector coefficients
   • y_i ∈ {-1, +1} are class labels (one-vs-rest for multi-class)
   
   Hyperparameters optimized:
   • C (regularization parameter): Controls margin tolerance [0.1, 1, 10, 100]
   • γ (gamma, kernel parameter): Controls decision boundary curvature [0.001, 0.01, 0.1, 1]
   
   Hyperparameter Tuning Method: Grid Search with 5-fold Cross-Validation
   • Grid: 4 × 4 = 16 parameter combinations tested
   • Scoring metric: F1-score (weighted for multi-class imbalance)
   • Validation strategy: Stratified k-fold to preserve class distribution
   
   Training objective:
   Minimize: (1/2) ||w||² + C Σ ξ_i  (subject to margin constraints)
   
   where ξ_i are slack variables allowing margin violations.

MODULE 4: Controlled Noise Simulation Engine
   Purpose: Systematically inject degradation to test robustness
   
   Noise Models Applied:
   
   A. Gaussian Noise (signal degradation)
      Model: x_noisy(i,j) = x_clean(i,j) + η(i,j)
      where η(i,j) ~ N(0, σ²) is independently drawn at each pixel
      
      Noise Intensities:
      • Clean: σ = 0
      • Low: σ = 10 (in 8-bit scale: 0-255)
      • Medium: σ = 20
      • High: σ = 30
      • Severe: σ = 40
   
   B. Salt-and-Pepper Noise (pixel corruption)
      Model: For probability p:
      • Set p/2 of pixels to maximum value (white, 255)
      • Set p/2 of pixels to minimum value (black, 0)
      • Keep remaining 1-p pixels unchanged
      
      Probability Levels:
      • Clean: p = 0.00
      • Low: p = 0.05
      • Medium: p = 0.10
      • High: p = 0.15
      • Severe: p = 0.20
   
   Combined Application: Both noise types are applied sequentially to each image
   at each intensity level, creating 5 degradation conditions (Clean, Low, Medium, High, Severe).

MODULE 5: Testing and Evaluation Engine
   Purpose: Measure performance on noisy datasets
   
   Evaluation Metrics:
   
   • Accuracy = (TP + TN) / (TP + TN + FP + FN)
   • Precision = TP / (TP + FP)  [per-class: weighted by support]
   • Recall = TP / (TP + FN)     [per-class: weighted by support]
   • F1-score = 2(Precision × Recall) / (Precision + Recall)
   
   Evaluation Protocol:
   For each noise level (Clean, Low, Medium, High, Severe):
   1. Apply noise to test set images
   2. Extract HOG features from noisy images
   3. Predict class labels using trained SVM model
   4. Compute metrics against ground truth labels
   5. Record confusion matrix for error analysis

MODULE 6: Error Analysis Engine
   Purpose: Understand failure modes and identify vulnerable character pairs
   
   Analysis Components:
   
   • Per-class confusion analysis: Which characters are most misclassified?
   • Confused character pairs: Which character pairs are most frequently confused?
   • False positive/negative analysis: Type of errors (false alarms vs missed detections)
   
   Output: Confusion matrix visualization and error distribution plots

MODULE 7: Visualization Dashboard
   Purpose: Generate publication-ready figures showing robustness findings
   
   Key Visualizations:
   • Accuracy vs Noise Level: Bar chart showing all metrics across noise intensities
   • Degradation Curve: Line plot showing accuracy decline with increasing noise
   • Confusion Matrices: Heatmaps for each noise condition
   • Error Distribution: Identifying most vulnerable character pairs

D. Feature Extraction Using Histogram of Oriented Gradients (HOG)

HOG Feature Overview:
   Histogram of Oriented Gradients (HOG) is a feature descriptor that captures local 
   shape information by computing gradient orientation distributions. It is widely adopted 
   for character recognition due to its robustness to local illumination changes.

HOG Computation Steps:

   Step 1 - Gradient Calculation:
   Compute image gradients using Sobel operators:
   
       G_x = [-1  0  1]     G_y = [-1 -2 -1]
             [-2  0  2]           [ 0  0  0]
             [-1  0  1]           [ 1  2  1]
   
   For each pixel (i,j):
       G_x(i,j) = image(i,j+1) - image(i,j-1)  [approximate]
       G_y(i,j) = image(i+1,j) - image(i-1,j)  [approximate]
   
   Step 2 - Gradient Magnitude and Orientation:
       mag(i,j) = √(G_x² + G_y²)
       θ(i,j) = arctan(G_y / G_x)  [in range [0°, 180°)]
   
   Step 3 - Cell-based Histogram Computation:
   Divide image into 8×8 cells (for 64×64 image: 8×8 = 64 cells)
   For each cell, compute 9-bin histogram of gradient orientations:
   
       bins: [0°, 20°, 40°, 60°, 80°, 100°, 120°, 140°, 160°]
   
   For each pixel in cell:
       bin_index = round(θ / 20)  (unsigned orientation)
       histogram[bin_index] += magnitude
   
   Step 4 - Block Normalization:
   Group adjacent 2×2 cells into blocks (for 8×8 cells: 7×7 = 49 blocks with overlap)
   Normalize each block's concatenated histogram using L2-Hys:
   
       h_norm = h / √(||h||² + ε²)  [where ε prevents division by zero]
       h_clipped = min(h_norm, threshold)  [typical threshold = 0.2]
       h_final = h_clipped / √(||h_clipped||²)
   
   Step 5 - Feature Vector Assembly:
   Concatenate all normalized block histograms into single feature vector:
   
       Feature dimension = (cells_x - 1) × (cells_y - 1) × blocks² × bins
                         = 7 × 7 × 4 × 9
                         = 1764 features

Justification for HOG:
   • Robustness: HOG features are invariant to local photometric changes (lighting variation)
   • Character Detail: Captures edge information essential for distinguishing similar characters
   • Computational Efficiency: Significantly faster than deep learning features for this task
   • Proven Track Record: Extensively validated in handwritten character recognition literature

E. Classification Model: Support Vector Machine with RBF Kernel

SVM Fundamentals:

   SVM is a margin-based classifier that finds optimal decision boundaries in high-dimensional 
   feature space. For character recognition, SVM is particularly suited because:
   
   • Effective in high-dimensional spaces (1764 HOG features)
   • Strong generalization via margin maximization
   • Handles multi-class problems through one-vs-rest strategy
   • Limited by small training set size compared to deep learning

RBF Kernel Selection:

   The Radial Basis Function (RBF) kernel is chosen over linear kernels because 
   Baybayin characters exhibit non-linear separability in HOG feature space.
   
   RBF Kernel Definition:
       K(x_i, x_j) = exp(-γ ||x_i - x_j||²)
   
   where:
   • γ (gamma) controls the influence radius
     - High γ: tight decision boundaries, possible overfitting
     - Low γ: smoother boundaries, possible underfitting
   
   • ||x_i - x_j||² is the squared Euclidean distance in feature space

Multi-Class Extension (One-vs-Rest):

   For 14 Baybayin character classes, SVM is extended using one-vs-rest:
   • Train K binary classifiers (K = 14)
   • Each classifier separates one character from all others
   • Final prediction: class with highest decision function value

F. Hyperparameter Tuning via Grid Search and Cross-Validation

Purpose of Hyperparameter Tuning:

   SVM performance is highly sensitive to hyperparameter selection. Improper tuning leads to:
   • Overfitting (high C, high γ): Excellent training accuracy, poor robustness
   • Underfitting (low C, low γ): Poor accuracy on both clean and noisy data
   
   Goal: Find the "sweet spot" that maximizes robustness without sacrificing accuracy.

Grid Search Method:

   Parameter Space:
   • C ∈ {0.1, 1, 10, 100} (4 values)
   • γ ∈ {0.001, 0.01, 0.1, 1} (4 values)
   • Total combinations: 4 × 4 = 16
   
   Search Process:
   For each combination (C, γ):
       1. Train SVM with this parameter pair on training set
       2. Evaluate on validation set using 5-fold cross-validation
       3. Record F1-score (weighted for multi-class)
       4. Store results for comparison
   
   Selection Criterion:
       (C_best, γ_best) = argmax F1-score over all 16 combinations

Why F1-Score for Tuning:

   • Accuracy alone is misleading if classes are imbalanced
   • F1-score balances precision and recall:
       F1 = 2(Precision × Recall) / (Precision + Recall)
   • Weighted F1-score averages per-class F1 scores by class support

5-Fold Cross-Validation:

   Divide training data into 5 equal folds:
   
   For each fold:
       Train on 4 folds
       Validate on 1 fold
       Record F1-score
   
   Final CV score = mean of 5 fold scores (provides robustness estimate)

G. Controlled Noise Simulation (Core Contribution)

Noise Models:

   This study introduces TWO complementary noise models to simulate real-world degradation:

   1. GAUSSIAN NOISE (Signal-Level Degradation)
      Represents: Sensor noise, lighting variations, camera imperfections
      
      Mathematical Model:
          x_noisy(i,j) = x_clean(i,j) + η(i,j)
          where η(i,j) ~ N(0, σ²), independently at each pixel
      
      Characteristics:
      • Smooth degradation
      • Increases visual blur
      • Affects all pixels uniformly
      
      Intensity Levels (in 0-255 pixel value scale):
      • Clean: σ = 0
      • Low (10% corruption): σ = 10
      • Medium (20% corruption): σ = 20
      • High (30% corruption): σ = 30
      • Severe (40% corruption): σ = 40

   2. SALT-AND-PEPPER NOISE (Pixel-Level Corruption)
      Represents: Scanning artifacts, dust on paper, transmission errors
      
      Mathematical Model:
          For each pixel with probability p:
          • Set to max value (255, "salt") with probability p/2
          • Set to min value (0, "pepper") with probability p/2
          • Keep original value with probability 1-p
      
      Characteristics:
      • Abrupt pixel changes
      • Creates black-and-white "speckles"
      • More disruptive to character structure
      
      Probability Levels:
      • Clean: p = 0.00 (0% noise)
      • Low: p = 0.05 (5% pixels corrupted)
      • Medium: p = 0.10 (10% pixels corrupted)
      • High: p = 0.15 (15% pixels corrupted)
      • Severe: p = 0.20 (20% pixels corrupted)

Combined Noise Application:

   To simulate realistic degradation, both noise types are applied sequentially:
   
   Process:
   1. Start with clean image x_clean
   2. Add Gaussian noise: x_gauss = x_clean + N(0, σ²)
   3. Add salt-pepper to result: x_final = AddSaltPepper(x_gauss, p)
   4. Clip to valid range [0, 1]
   
   Justification:
   • Real-world images suffer from multiple degradation sources simultaneously
   �� Tests whether model robustness is additive or multiplicative

H. Testing and Evaluation Protocol

Evaluation Strategy:

   The model trained on CLEAN data is tested on FIVE degradation conditions to measure 
   robustness WITHOUT retraining.
   
   This design choice is critical because:
   ✓ Isolates noise robustness (not model adaptation)
   ✓ Reflects real-world deployment (model doesn't see test noise during training)
   ✓ Measures true generalization capability

Test Procedure for Each Noise Level:

   Input: Clean test images X_test (15% of total samples)
   
   Step 1 - Add Noise
       X_noisy = ApplyNoise(X_test, noise_config)
   
   Step 2 - Extract HOG Features
       F_noisy = ExtractHOG(X_noisy)
   
   Step 3 - Normalize Features
       F_normalized = (F_noisy - mean_train) / std_train
       [Using statistics computed on clean training data]
   
   Step 4 - Predict Classes
       y_pred = model.predict(F_normalized)
   
   Step 5 - Compute Metrics
       Accuracy = mean(y_pred == y_test)
       Precision = TP / (TP + FP)  [per-class, weighted]
       Recall = TP / (TP + FN)      [per-class, weighted]
       F1 = 2(Precision × Recall) / (Precision + Recall)  [weighted]

Performance Metrics Explained:

   • Accuracy: Proportion of correct predictions
     - Easy to interpret but can be misleading with imbalanced classes
     - PRIMARY metric for this study (classes are balanced)
   
   • Precision: Of predicted positive, how many are actually positive?
     - Penalizes false positives (misidentifying another character as target)
   
   • Recall: Of actual positive, how many did we find?
     - Penalizes false negatives (missing target character)
   
   • F1-score: Harmonic mean of precision and recall
     - Balances both types of errors

I. Error Analysis and Confusion Characterization

Confusion Matrix Analysis:

   For each noise condition, a 14×14 confusion matrix is computed:
   
       CM[i,j] = number of samples of class i predicted as class j
   
   Diagonal elements CM[i,i] represent correct classifications.
   Off-diagonal elements CM[i,j] represent errors.

Per-Class Error Metrics:

   For each Baybayin character class i:
   
   • True Positives (TP_i): Correctly identified samples of class i
   • False Positives (FP_i): Incorrect samples predicted as class i
   • False Negatives (FN_i): Samples of class i predicted as other classes
   
   Per-class metrics:
   • Precision_i = TP_i / (TP_i + FP_i)
   • Recall_i = TP_i / (TP_i + FN_i)
   • F1_i = 2(Precision_i × Recall_i) / (Precision_i + Recall_i)

Character Pair Confusion Analysis:

   Identify most frequently confused character pairs by examining off-diagonal peaks 
   in confusion matrix.
   
   Ranking: Sort all CM[i,j] (i ≠ j) in descending order
   
   Questions addressed:
   • Are visually similar characters (e.g., BA vs PA) more frequently confused?
   • Does this confusion increase with noise?
   • Which character pair shows the steepest degradation?
