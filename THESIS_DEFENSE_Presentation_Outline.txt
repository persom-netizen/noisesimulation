SLIDE 4: Support Vector Machines (SVM) Fundamentals (1.5 min)

Title: Why SVM? Classification via Margin Maximization

Key Concept:
SVM finds an OPTIMAL DECISION BOUNDARY that maximizes the margin 
between different classes.

Visual: 2D diagram showing SVM decision boundary with support vectors

Mathematical Intuition:
"Find a hyperplane that best separates classes in feature space"

Why SVM for Character Recognition:
вң“ Effective with high-dimensional data (1764 HOG features)
вң“ Strong generalization (margin principle prevents overfitting)
вң“ Computationally efficient compared to deep learning
вң“ Proven track record in handwritten digit recognition

Kernel Trick - RBF Kernel:
K(x_i, x_j) = exp(-Оі ||x_i - x_j||ВІ)

Interpretation: Non-linear decision boundaries allow curved separators 
(important for handwritten characters which have curved shapes)

Speaker Notes:
"SVM's key innovation is the kernel trickвҖ”it allows us to create curved decision 
boundaries without explicitly computing high-dimensional transformations. For curved 
characters like Baybayin, this is essential."

SLIDE 5: HOG Feature Extraction (1.5 min)

Title: Histogram of Oriented Gradients (HOG)

Process Flow:

INPUT: Raw image (64Г—64 pixels)
        вҶ“
COMPUTE: Gradient direction at each pixel
        пҝҪпҝҪ
CELL HISTOGRAM: 8Г—8 cells, 9-bin orientation histograms
        вҶ“
BLOCK NORMALIZATION: L2-Hys normalization for stability
        вҶ“
OUTPUT: 1764-dimensional feature vector

Why HOG Works:
вҖў Captures EDGE patterns essential for character recognition
вҖў Invariant to lighting changes (uses gradient, not pixel value)
вҖў Computationally efficient
вҖў Well-established in computer vision literature

Example Interpretation:
Character "BA" in HOG space = "lots of vertical edges + curved strokes"
Character "CA" in HOG space = "curved strokes at different angles"
вҶ’ These form separable clusters in 1764-dimensional feature space

Visual: Sample character вҶ’ gradient map вҶ’ HOG visualization

Speaker Notes:
"Think of HOG as asking: 'In which directions are the edges in this image?' 
By counting these directions in small regions, we capture the shape structure 
without being fooled by lighting variations."

SLIDE 7: Noise Models - Real-World Degradation (1 min)

Title: How Do Real Images Get Damaged?

Two Noise Types in Our Study:

1. GAUSSIAN NOISE (Signal degradation)
   Model: x_noisy = x_clean + N(0, ПғВІ)
   Real-world cause: Sensor noise, lighting variations, camera imperfections
   Effect: Smooth degradation, overall blur
   
   Visual: Clean image вҶ’ slightly noisy вҶ’ moderately noisy вҶ’ severely noisy

2. SALT-AND-PEPPER NOISE (Pixel corruption)
   Model: Random p% of pixels set to black or white
   Real-world cause: Dust on scanner, transmission errors, scanning artifacts
   Effect: Abrupt black/white speckles, structural disruption
   
   Visual: Clean image вҶ’ with speckles вҶ’ heavily speckled

Why Both?
Real documents suffer from BOTH types simultaneously.
Testing both isolates their individual and combined effects.

Speaker Notes:
"These aren't arbitrary choicesвҖ”they represent real degradation mechanisms that 
happen to physical documents during scanning or photography."

SLIDE 9: Hyperparameter Tuning - Grid Search (1.5 min)

Title: Finding Optimal SVM Settings

The Challenge:
SVM has two critical parameters:
вҖў C (regularization): Controls margin strictness
  - Low C: Flexible boundaries, possible underfitting
  - High C: Strict boundaries, possible overfitting
  
вҖў Оі (gamma): Controls RBF kernel influence
  - Low Оі: Smooth boundaries
  - High Оі: Complex boundaries

Grid Search Strategy:

C values: {0.1, 1, 10, 100}
Оі values: {0.001, 0.01, 0.1, 1}
Total combinations: 16

Process:
For each (C, Оі) pair:
  вҶ’ Train SVM on 70% data
  вҶ’ Validate on 5-fold cross-validation
  вҶ’ Score using F1-score (weighted for multi-class)
  вҶ’ Record best combination

Result: (C=10, Оі=0.01) selected as optimal

Why F1-Score?
Balances precision (avoid false alarms) and recall (don't miss characters)
More informative than raw accuracy alone

Speaker Notes:
"Grid search is methodical but thorough. We test all 16 combinations and choose 
the one with the best validated performance. This prevents lucky selections."

SLIDE 11: Train-Test Separation Strategy (1 min)

Title: Why We NEVER Train on Noisy Data

Critical Design Principle:

WRONG APPROACH (That Many Papers Do):
Train on clean images
Train on noisy images (at every level)
Report accuracy on each
вҶ’ PROBLEM: Model adapts to noise, not truly robust

OUR APPROACH (Correct):
Train ONCE on clean images
Test on: Clean, Low, Medium, High, Severe (SAME model)
вҶ’ BENEFIT: Measures true generalization, simulates real deployment

Justification:
"In production, you train on curated, clean data (or labeled clean data).
At test time, you encounter degraded real-world images.
Can your model handle it?"

Validation:
The model's hyperparameters (C, Оі) were optimized on CLEAN data.
We then test whether those parameters remain optimal under noise.

Speaker Notes:
"This is the hard truth of robustness testing. You can't cheat by training on 
noisy dataвҖ”that's not how real systems work."

SLIDE 12: Evaluation Metrics (1 min)

Title: How We Measure Performance

Four Metrics Computed at Each Noise Level:

1. ACCURACY
   Definition: (TP + TN) / Total
   Interpretation: Overall correctness rate
   Range: 0-100%

2. PRECISION
   Definition: TP / (TP + FP)
   Interpretation: Of predicted positives, how many are correct?
   Penalizes false alarms

3. RECALL
   Definition: TP / (TP + FN)
   Interpretation: Of actual positives, how many did we find?
   Penalizes missed detections

4. F1-SCORE
   Definition: 2 Г— (Precision Г— Recall) / (Precision + Recall)
   Interpretation: Harmonic meanвҖ”balances precision and recall
   Most informative single metric

All metrics are WEIGHTED by class support (handles any class imbalance)

Expected Behavior:
As noise increases:
  Accuracy: Steadily decreases
  Precision: May decrease more than Recall (more false positives under noise)
  F1-Score: Decreases (combined effect)

Speaker Notes:
"Different metrics tell different stories. Accuracy alone can be misleading, 
especially if one character is harder than others. The weighted F1-score gives 
us the most honest picture."

SLIDE 18: Why These Results Matter (1.5 min)

Title: Interpreting the Robustness Findings

Central Finding: THREE-PHASE DEGRADATION

PHASE 1 (0-5% noise): GRACEFUL DEGRADATION
  Accuracy > 90%
  Character pairs remain separable
  RBF kernel maintains decision boundary integrity
  вҶ’ SAFE for deployment

PHASE 2 (5-15% noise): ACCELERATED FAILURE
  Accuracy drops from 90% to 73%
  Character pair confusion increases exponentially
  False positives increase more than false negatives
  вҶ’ RISKY for deployment, need additional measures

PHASE 3 (15-20% noise): CRITICAL FAILURE
  Accuracy approaches random (7.1% for 14 classes)
  Model loses discrimination ability
  вҶ’ NOT SUITABLE for deployment

Mechanism:
Why does degradation accelerate?

HOG Feature Space Under Noise:
вҖў Clean: Classes form well-separated clusters
вҖў Low noise: Clusters expand slightly
вҖў Medium noise: Clusters begin overlapping (confusion starts)
вҖў High noise: Clusters merge, decision boundaries collapse
вҖў Severe: Near-uniform distribution (random guessing)

Mathematical Explanation:
RBF kernel: K(x_i, x_j) = exp(-Оі ||x_i - x_j||ВІ)

Under noise:
вҖў Feature vectors become less reliable
вҖў Distance calculations become noisy
вҖў Kernel values saturate
вҖў Support vectors lose discriminative power

how RBF kernels workвҖ”they're sensitive to feature reliability, and noise

    вҖў Unweighted: (68 + 78) / 2 = 73%
